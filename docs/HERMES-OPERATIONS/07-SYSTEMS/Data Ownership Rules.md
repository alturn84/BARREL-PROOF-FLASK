# Data Ownership Rules
**Version:** 1.0
**Last Modified:** 2025-06-06

---

## Purpose

This document defines who owns what data within the Barrel Proof system and what Hermes is and is not authorized to do with each data type.

---

## Data Ownership Map

| Data Type | Owner | Hermes Authorization |
|-----------|-------|---------------------|
| MLB game data (raw API) | External (MLB API) | Read only — fetch and parse |
| Game card JSON | Pipeline output | Read and write within workspace |
| Standings JSON | Pipeline output | Read and write within workspace |
| Schedule JSON | Pipeline output | Read and write within workspace |
| Player stats JSON | Pipeline output | Read and write within workspace |
| Generated content (summaries, Press Box, etc.) | Barrel Proof | Read and write within workspace |
| Production codebase | Barrel Proof / Allan Turner | **Read only** — Hermes does not modify code |
| GitHub repository history | Barrel Proof / Allan Turner | Commit and push pipeline output only |
| Render configuration | Barrel Proof / Allan Turner | **No access** — managed by Allan |
| Environment variables / API keys | Allan Turner | **No access** — Hermes reads from environment, does not manage |
| Barrel Proof Vault (Obsidian) | Allan Turner | **No access** — separate system |

---

## File Modification Rules

Hermes may create and modify:
- Data JSON files generated by Cron Job 1
- Content files generated by Cron Job 2
- Log files within its operational scope

Hermes must not modify:
- Application code files (`.py`, `.js`, `.html`, `.css`, etc.)
- Configuration files (`render.yaml`, `requirements.txt`, etc.)
- `.gitignore` or any repository configuration
- Files outside `/workspace/barrel-proof`

---

## Commit Authorization

Every morning commit contains only:
- Files generated by the pipeline in the current run
- Log updates

Hermes never commits:
- Code modifications
- Configuration changes
- Files from outside the workspace
- Unvalidated data
