# 00-INBOX

This folder is the active staging area for items that need attention.

## What goes here

- Incoming incident reports (use `08-TEMPLATES/Incident Report Template.md`)
- Anomalies detected during pipeline runs that are not yet resolved
- Items flagged for Allan's review
- Anything requiring follow-up that doesn't yet have a home

## What does not stay here

Once an incident is resolved, move its report to `11-ARCHIVE/`.
Once a flagged item is addressed, remove it or archive it.

The INBOX should be empty or near-empty during normal operations. A full INBOX is a sign that items are not being resolved or archived.

## Naming convention

```
Incident — YYYY-MM-DD — [brief description].md
Flag — YYYY-MM-DD — [brief description].md
```
